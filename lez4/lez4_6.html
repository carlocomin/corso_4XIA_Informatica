Incapsulamento forte e metodi const

Obiettivo della pagina

Capire come si difende davvero l'invariante di una classe durante tutta la sua vita, non solo nel costruttore.

Idea chiave: incapsulamento forte significa che lo stato si modifica solo passando da metodi che applicano le regole del tipo.

Dati privati, regole centralizzate

Se i campi sono pubblici, chiunque puo mettere l'oggetto in uno stato assurdo. Se invece i campi sono privati, tutte le modifiche passano da un punto controllato.

class Orario {
private:
    int hh_;
    int mm_;
public:
    bool set(int h, int m){
        if (0 <= h && h < 24 && 0 <= m && m < 60){
            hh_ = h;
            mm_ = m;
            return true;
        }
        return false;
    }
};

Che cosa vuol dire const

Un metodo const promette di non modificare lo stato osservabile dell'oggetto.

  • e il modo corretto per marcare osservatori e query;
  • rende leggibile l'intento del metodo;
  • permette di usare il metodo anche su oggetti const.
double saldo() const { return saldo_; }

Getter e setter non sono automatici

Non ogni campo privato richiede per forza un getter e un setter meccanico.

  • un getter ha senso se il dato serve davvero all'esterno;
  • un setter ha senso solo se esiste una modifica coerente con il modello;
  • spesso e meglio un metodo di dominio come versa, preleva o scala.
Errore tipico: mettere tutto private e poi riesporre ogni campo con getter e setter banali. Formalmente hai incapsulato, ma di fatto hai riaperto tutto.
