Utility e input robusto

Obiettivo della pagina

Passare da funzioni minime a funzioni davvero riusabili: azzerare, riempire, stampare con una forma leggibile e non bloccarsi al primo input sbagliato.

Idea chiave: una piccola libreria di utility semplici fa risparmiare piu tempo di quanto faccia una lunga demo nel main.

Tre utility che conviene avere subito

void azzera(int (&a)[R][C]){
  for (int r = 0; r < R; ++r)
    for (int c = 0; c < C; ++c)
      a[r][c] = 0;
}

void riempiCostante(int (&a)[R][C], int v){
  for (int r = 0; r < R; ++r)
    for (int c = 0; c < C; ++c)
      a[r][c] = v;
}

Non sembrano funzioni "importanti", ma servono continuamente in test, esercizi e debugging.

Rendere la stampa piu leggibile

Una matrice stampata bene aiuta a capire subito se il programma sta facendo quello che deve fare. Se vuoi una forma un po' piu ordinata, puoi introdurre intestazioni o larghezze fisse.

void stampaConIndici(const int (&a)[R][C]){
  for (int r = 0; r < R; ++r){
    std::cout << r << ": ";
    for (int c = 0; c < C; ++c)
      std::cout << a[r][c] << ' ';
    std::cout << '\n';
  }
}

Questo e gia sufficiente per molti esercizi scolastici: non serve sempre una tabella perfetta, serve soprattutto una stampa coerente.

Input robusto: che cosa significa davvero

Significa non assumere che l'utente scriva sempre il dato corretto al primo colpo. In un esercizio iniziale puo bastare controllare l'esito della lettura; in un programma un po' piu serio conviene anche ripulire lo stream e ripetere la richiesta.

while (!(std::cin >> a[r][c])) {
  std::cin.clear();
  std::cin.ignore(100000, '\n');
  std::cout << "Valore non valido, riprova: ";
}
Regola pratica: separa sempre la logica della matrice dalla logica di interazione con l'utente. Una funzione che "calcola" non dovrebbe anche chiedere input o stampare messaggi casuali.
