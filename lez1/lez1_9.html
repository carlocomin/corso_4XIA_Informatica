Firme corrette in C++ e C

Obiettivo della pagina

Tradurre il modello mentale della pagina precedente in firme corrette e leggibili.

Idea chiave: in C++ conviene sfruttare il tipo in modo forte; in C bisogna rendere esplicita almeno la seconda dimensione o usare VLA quando il contesto lo permette.

C++: riferimento a matrice o template

constexpr int R = 3;
constexpr int C = 4;

void stampa(const int (&m)[R][C]){
  for (int r = 0; r < R; ++r)
    for (int c = 0; c < C; ++c)
      std::cout << m[r][c] << ' ';
}

Se vuoi piu flessibilita, puoi usare un template e lasciare al compilatore la deduzione di R e C.

C: seconda dimensione nota o VLA

#define C 4

void stampa(int R, const int m[][C]){
  for (int r = 0; r < R; ++r)
    for (int c = 0; c < C; ++c)
      printf("%d ", m[r][c]);
}

In C99 puoi anche usare VLA e quindi passare le dimensioni a runtime:

void stampa_vla(int R, int C, const int m[R][C]) { /* ... */ }

Questo e corretto in C, ma non va trattato come modello standard di C++.

Firme da evitare

  • void f(int **m) per una matrice statica int a[R][C];
  • firme che nascondono il significato delle dimensioni;
  • soluzioni "funzionano nel mio compilatore" ma non sono didatticamente pulite o portabili.

Regola pratica: se la matrice e statica e contigua, la firma deve riflettere proprio questo fatto, non un'altra struttura dati.

Passo successivo. Nella L1.10 usciamo dal caso statico e affrontiamo il problema vero delle dimensioni note solo a runtime: matrici dinamiche, contiguita e scelte di rappresentazione.
