Inizializzare bene in C++

Obiettivo della pagina

Capire come far nascere una struct in uno stato sensato e sicuro.

Idea chiave: in C++ le {} non sono solo una convenzione elegante: sono spesso il modo piu chiaro e piu sicuro di inizializzare una struct.

Valori indeterminati vs stato noto

struct Studente {
    std::string nome;
    std::string cognome;
    int eta;
    double media;
};

Studente a;   // eta e media non sono automaticamente azzerati
Studente b{}; // eta = 0, media = 0.0, stringhe vuote

Il punto importante non e ricordare un dettaglio sintattico, ma evitare di leggere campi fondamentali prima di aver dato loro un valore sensato.

Aggregate initialization

Studente s{"Ada", "Lovelace", 17, 9.4};

Quando usi una lista di inizializzazione per una struct semplice, i valori seguono l'ordine dei campi. Questo e potente, ma richiede disciplina: se sbagli ordine, costruisci un bug silenzioso.

Default dei campi

struct Studente {
    std::string nome = "Anon";
    std::string cognome = "N/A";
    int eta = 0;
    double media = 0.0;
};

Questa scelta e utile quando vuoi che la struct abbia un valore iniziale ragionevole anche senza specificare tutti i campi.

Braces e narrowing

int x(3.7);  // conversione implicita a 3
int y{3.7}; // errore: narrowing

Le {} rendono piu difficile introdurre conversioni pericolose senza accorgersene. Per questo, nel corso, restano la forma preferita.
