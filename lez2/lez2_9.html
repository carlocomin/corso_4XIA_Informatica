Lambda, ordinamento e filtro

Obiettivo della pagina

Usare le collezioni di record in modo piu intelligente: ordinare, cercare, contare, filtrare.

Idea chiave: le lambda permettono di scrivere il criterio di confronto o di selezione accanto all'algoritmo che lo usa. Questo rende il codice piu locale e piu leggibile.

Che cos'e una lambda

auto promosso = [](const Studente& s){
    return s.media >= 6.0;
};

Pensala come una piccola funzione anonima definita nel punto esatto in cui serve. Non sostituisce tutte le funzioni normali, ma e perfetta per criteri brevi e localizzati.

Ordinare una classe di studenti

std::sort(classe.begin(), classe.end(),
    [](const Studente& a, const Studente& b){
        return a.cognome < b.cognome;
    });

Qui la lambda descrive il criterio. L'algoritmo si occupa dell'ordinamento, la lambda si occupa del significato dell'ordine.

Ricerca e filtro

auto it = std::find_if(classe.begin(), classe.end(),
    [](const Studente& s){
        return s.media < 6.0;
    });

int quanti = std::count_if(classe.begin(), classe.end(),
    [](const Studente& s){
        return s.eta >= 18;
    });

La domanda didattica giusta qui non e "ricordo il nome della funzione STL?", ma "so esprimere una proprieta chiara su un record?".

Confronto minimo con il C

Nel C puro non hai lambda standard. Lo stesso tipo di problema si affronta con funzioni di confronto esplicite, per esempio insieme a qsort. Il concetto pero resta lo stesso: separare l'algoritmo dal criterio.

Errore tipico: scrivere lambda troppo opache, con catture inutili o confronti incoerenti. Se il criterio diventa lungo o difficile da leggere, probabilmente e meglio estrarlo in una funzione dedicata.
