Inizializzare in C

Obiettivo della pagina

Capire come inizializzare bene una struct in C, senza trasferire in modo automatico abitudini del C++ che qui non esistono.

Idea chiave: in C le struct si inizializzano bene, ma bisogna essere piu espliciti, soprattutto quando compaiono array di char e designated initializers.

Struct d'esempio

typedef struct {
    char nome[32];
    char cognome[32];
    int eta;
    double media;
} Studente;

Qui non hai std::string, quindi le stringhe sono buffer di caratteri con terminatore '\0'. Questo cambia il modo in cui ragioni su capienza e inizializzazione.

Lista in ordine dei campi

Studente s = { "Ada", "Lovelace", 17, 9.4 };

Come in C++, l'ordine conta. Ma qui il messaggio didattico importante e un altro: le stringhe vengono copiate dentro gli array della struct, non gestite da un oggetto dinamico.

Designated initializers

Studente s = {
    .cognome = "Lovelace",
    .nome = "Ada",
    .media = 9.4,
    .eta = 17
};

Questa forma e didatticamente molto utile in C, perche rende il codice meno fragile rispetto all'ordine dei campi.

Inizializzazione parziale e azzeramento

Studente s1 = { "Grace" };
Studente s2 = {0};

{0} e una scorciatoia importante: ti permette di partire da una struct completamente azzerata. In C, per le variabili automatiche, questo e spesso il modo piu sicuro per evitare valori spazzatura.

Errore tipico: trattare le stringhe C come se fossero oggetti elastiche come std::string. Qui serve sempre attenzione alla capienza del buffer.
