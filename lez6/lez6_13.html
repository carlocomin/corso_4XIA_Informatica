Esercizio pratico finale
Mini BST completo: costruzione, visite, ricerca, cancellazione

Obiettivo dell'esercizio pratico

Qui la teoria sugli alberi deve diventare una struttura dati completa e realmente usabile. Non basta più conoscere definizioni e casi classici: bisogna progettare un piccolo BST che sappia gestire memoria, inserimento, ricerca, cancellazione, visite e alcune interrogazioni utili come altezza e numero di nodi.

Questo laboratorio è molto formativo perché costringe a tenere insieme quasi tutto il percorso della lezione: rappresentazione del nodo, ricorsione sui sottoalberi, visite, proprietà del BST, casi della cancellazione, dipendenza del costo dall'altezza e attenzione alla memoria dinamica.

Idea chiave: un buon laboratorio finale non mostra solo che sai scrivere funzioni separate, ma che sai farle convivere in un piccolo progetto coerente.

Testo dell'esercizio

Implementa un mini albero binario di ricerca di interi in C++ con le seguenti funzionalità:

  1. struttura del nodo con chiave intera e puntatori ai figli;
  2. creazione e distruzione corretta dell'albero;
  3. inserimento di una chiave;
  4. ricerca di una chiave;
  5. visite preorder, inorder e postorder;
  6. stampa per livelli con BFS;
  7. cancellazione di una chiave;
  8. calcolo di altezza e numero totale dei nodi.

Vincoli consigliati:

  • evita variabili globali;
  • usa nomi chiari e funzioni corte;
  • gestisci in modo ordinato la memoria dinamica;
  • scegli e documenta una policy per i duplicati. In questa soluzione useremo: duplicati ignorati.

Bonus:

  • funzione che segnala se l'albero è fortemente sbilanciato;
  • bozza di rotateLeft e rotateRight con spiegazione;
  • piccolo menu testuale nel main() oppure dataset iniziale di prova.

Strategia progettuale prima del codice

La scelta più pulita, a questo livello, è racchiudere l'ownership dell'albero in una piccola classe BST che possiede un solo puntatore membro: la radice. In questo modo il resto del programma non deve conoscere tutti i dettagli della memoria dinamica.

Le funzioni ricorsive più comode possono restare private e lavorare su un Node* o const Node*. L'interfaccia pubblica, invece, espone operazioni semplici come insert, contains, remove, printInorder, height e countNodes.

Messaggio metodologico: il laboratorio non richiede per forza C++ "super moderno" con template e astrazioni sofisticate. Qui conta soprattutto una struttura chiara, una buona separazione tra interfaccia pubblica e supporto ricorsivo privato, e una gestione corretta dei nodi.

Che cosa osservare nella soluzione

  • Ownership centralizzata: il BST possiede la radice e distrugge tutto nell'apposito distruttore.
  • Ricorsione mirata: le funzioni private lavorano sui sottoalberi come unità autonome.
  • Cancellazione completa: i tre casi classici sono trattati in modo distinto.
  • Visite diverse, problemi diversi: inorder per l'ordine, postorder per la distruzione, BFS per i livelli.
  • Bonus coerente: lo squilibrio viene trattato come estensione naturale del tema dell'altezza.

Soluzione guidata

Prova prima a progettare da solo la struttura della classe e le firme dei metodi. Poi apri la soluzione e controlla soprattutto se la logica dei sottoalberi è chiara e se i casi della cancellazione sono leggibili.

#include <algorithm>
#include <iostream>
#include <queue>
#include <vector>

class BST {
private:
    struct Node {
        int key;
        Node* left;
        Node* right;
    };

    Node* root_ = nullptr;

    static Node* createNode(int key) {
        return new Node{key, nullptr, nullptr};
    }

    static void destroy(Node* p) {
        if (p == nullptr) return;
        destroy(p->left);
        destroy(p->right);
        delete p;
    }

    static Node* insertRec(Node* p, int key) {
        if (p == nullptr) return createNode(key);

        if (key < p->key) {
            p->left = insertRec(p->left, key);
        } else if (key > p->key) {
            p->right = insertRec(p->right, key);
        }
        return p; // duplicato ignorato
    }

    static bool containsRec(const Node* p, int key) {
        if (p == nullptr) return false;
        if (key == p->key) return true;
        if (key < p->key) return containsRec(p->left, key);
        return containsRec(p->right, key);
    }

    static Node* findMin(Node* p) {
        while (p != nullptr && p->left != nullptr) p = p->left;
        return p;
    }

    static Node* removeRec(Node* p, int key) {
        if (p == nullptr) return nullptr;

        if (key < p->key) {
            p->left = removeRec(p->left, key);
        } else if (key > p->key) {
            p->right = removeRec(p->right, key);
        } else {
            if (p->left == nullptr && p->right == nullptr) {
                delete p;
                return nullptr;
            }

            if (p->left == nullptr) {
                Node* tmp = p->right;
                delete p;
                return tmp;
            }

            if (p->right == nullptr) {
                Node* tmp = p->left;
                delete p;
                return tmp;
            }

            Node* succ = findMin(p->right);
            p->key = succ->key;
            p->right = removeRec(p->right, succ->key);
        }

        return p;
    }

    static void preorderRec(const Node* p) {
        if (p == nullptr) return;
        std::cout << p->key << ' ';
        preorderRec(p->left);
        preorderRec(p->right);
    }

    static void inorderRec(const Node* p) {
        if (p == nullptr) return;
        inorderRec(p->left);
        std::cout << p->key << ' ';
        inorderRec(p->right);
    }

    static void postorderRec(const Node* p) {
        if (p == nullptr) return;
        postorderRec(p->left);
        postorderRec(p->right);
        std::cout << p->key << ' ';
    }

    static int heightRec(const Node* p) {
        if (p == nullptr) return 0;
        return 1 + std::max(heightRec(p->left), heightRec(p->right));
    }

    static int countRec(const Node* p) {
        if (p == nullptr) return 0;
        return 1 + countRec(p->left) + countRec(p->right);
    }

public:
    ~BST() {
        destroy(root_);
    }

    void insert(int key) {
        root_ = insertRec(root_, key);
    }

    bool contains(int key) const {
        return containsRec(root_, key);
    }

    void remove(int key) {
        root_ = removeRec(root_, key);
    }

    void printPreorder() const {
        preorderRec(root_);
        std::cout << '\n';
    }

    void printInorder() const {
        inorderRec(root_);
        std::cout << '\n';
    }

    void printPostorder() const {
        postorderRec(root_);
        std::cout << '\n';
    }

    void printLevelOrder() const {
        if (root_ == nullptr) {
            std::cout << "(albero vuoto)\n";
            return;
        }

        std::queue<const Node*> q;
        q.push(root_);

        while (!q.empty()) {
            const Node* p = q.front();
            q.pop();

            std::cout << p->key << ' ';

            if (p->left != nullptr) q.push(p->left);
            if (p->right != nullptr) q.push(p->right);
        }
        std::cout << '\n';
    }

    int height() const {
        return heightRec(root_);
    }

    int countNodes() const {
        return countRec(root_);
    }
};

int main() {
    BST t;

    std::vector<int> valori{10, 5, 18, 2, 7, 12, 25, 15};
    for (int x : valori) t.insert(x);

    std::cout << "Inorder:    ";
    t.printInorder();

    std::cout << "Preorder:   ";
    t.printPreorder();

    std::cout << "Postorder:  ";
    t.printPostorder();

    std::cout << "Per livelli:";
    t.printLevelOrder();

    std::cout << "Contiene 12? " << (t.contains(12) ? "si" : "no") << '\n';
    std::cout << "Contiene 99? " << (t.contains(99) ? "si" : "no") << '\n';

    std::cout << "Altezza = " << t.height() << '\n';
    std::cout << "Numero nodi = " << t.countNodes() << '\n';

    t.remove(10); // caso con due figli
    std::cout << "\nDopo remove(10), inorder: ";
    t.printInorder();
}

Perché questa soluzione è buona: l'ownership dei nodi è concentrata dentro la classe BST, le operazioni pubbliche sono pulite, e le funzioni ricorsive private lavorano in modo uniforme su qualunque sottoalbero.

Osserva bene: distruzione, altezza, conteggio, visite, inserimento e cancellazione sono tutti esempi diversi della stessa idea di fondo: lavorare ricorsivamente su un nodo come radice di un sottoalbero.

Commento ragionato alla soluzione

La scelta di usare una classe BST con una struct Node interna è molto istruttiva. Permette di vedere chiaramente due livelli: da una parte l'interfaccia pubblica del contenitore, dall'altra le funzioni ricorsive che manipolano i dettagli strutturali.

Il distruttore chiama destroy(root_), quindi la memoria viene liberata in modo centralizzato e coerente. Questo è importante: anche se usiamo puntatori grezzi per i nodi, non stiamo lasciando la gestione della memoria "sparsa" nel programma.

La funzione removeRec è il cuore più delicato del progetto. Proprio per questo conviene leggerla separando la discesa nell'albero dai tre casi classici: foglia, un figlio, due figli. Se questa funzione è capita davvero, la struttura del BST è capita davvero.

Bonus 1 · Segnalare uno squilibrio forte

Come estensione coerente con la teoria finale, possiamo aggiungere una funzione che segnali se in qualche nodo la differenza tra altezza sinistra e destra supera una soglia. Non stiamo implementando un AVL completo, ma stiamo misurando il problema che motiva il bilanciamento.

static bool isVeryUnbalancedRec(const Node* p, int soglia = 2) {
    if (p == nullptr) return false;

    int hs = heightRec(p->left);
    int hd = heightRec(p->right);
    int diff = (hs > hd) ? (hs - hd) : (hd - hs);

    if (diff > soglia) return true;

    return isVeryUnbalancedRec(p->left, soglia) ||
           isVeryUnbalancedRec(p->right, soglia);
}

Messaggio didattico: non basta sapere che l'albero "può sbilanciarsi". Qui stiamo trasformando quell'idea in una proprietà controllabile dal programma.

Bonus 2 · Rotazioni locali

Come ulteriore estensione, si possono implementare le rotazioni locali già introdotte nella teoria. Non serve integrarle in un AVL completo: basta capire che una rotazione restituisce una nuova radice locale del sottoalbero e riconnette i puntatori senza perdere i sottoalberi intermedi.

static Node* rotateRight(Node* y) {
    if (y == nullptr || y->left == nullptr) return y;

    Node* x = y->left;
    Node* T2 = x->right;

    x->right = y;
    y->left = T2;

    return x;
}

static Node* rotateLeft(Node* x) {
    if (x == nullptr || x->right == nullptr) return x;

    Node* y = x->right;
    Node* T2 = y->left;

    y->left = x;
    x->right = T2;

    return y;
}

Che cosa c'è di importante qui: la rotazione non crea o distrugge nodi. Modifica soltanto la forma locale del sottoalbero, mantenendo però l'ordine del BST.

Riflessione finale: perché questa è una buona consegna

Questo laboratorio è didatticamente molto ricco perché costringe a tenere insieme aspetti spesso studiati separatamente: puntatori, ricorsione, strutture dati dinamiche, visite, scelta degli algoritmi e gestione della memoria. Non è solo "esercizio sugli alberi": è un piccolo progetto in cui le idee devono collaborare.

In più, il BST è una struttura ottima per imparare una lezione di metodo: conoscere la teoria di una struttura dati significa capire anche quando e perché le sue operazioni funzionano bene, e quando invece la forma dell'albero rischia di tradirne le promesse.

Checklist finale della lezione 6

  • sai rappresentare un albero binario con nodi e puntatori;
  • sai distinguere preorder, inorder, postorder, DFS e BFS;
  • sai spiegare perché l'inorder di un BST produce una sequenza ordinata;
  • sai implementare ricerca, inserimento e cancellazione di base;
  • sai motivare perché altezza e bilanciamento contano;
  • riconosci il ruolo delle rotazioni come correzioni locali della forma.

Se tutti questi punti sono chiari, allora la lezione sugli alberi non è stata solo letta: è stata assimilata in modo operativo.

Il passo successivo, fuori da questa unità, potrebbe essere approfondire alberi bilanciati completi oppure confrontare i BST con altre strutture dati dinamiche per insiemi ordinati.
