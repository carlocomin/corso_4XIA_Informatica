Indici per file binari

Obiettivo della pagina

Chiudere la lezione mostrando come formato binario, offset e seek si combinano in una struttura dati semplice ma molto potente.

Idea chiave: un indice e una tabella che associa una chiave a una posizione nel file. Cerchi nell'indice e poi fai un seek diretto nel file dati.

Il problema senza indice

Se il file contiene molti record e vuoi cercarne uno per chiave, la soluzione piu semplice e scorrere tutto il file dall'inizio.

Questa strategia e lineare: ogni ricerca puo costare moltissimo.

Come ragiona un indice

  • nel file dati tieni i record veri;
  • nell'indice tieni coppie del tipo chiave, offset;
  • se l'indice e ordinato, puoi usare ricerca binaria;
  • quando trovi l'offset, salti subito al record nel file dati.

Perche questa idea e didatticamente importante

Qui si vede bene la differenza tra file testuali e file binari:

  • nel testo ti orienti con righe e parser;
  • nel binario puoi progettare record e offset come oggetti di primo livello;
  • il seek smette di essere un dettaglio tecnico e diventa parte del modello dati.

Pseudocodice minimale

cerca chiave nell'indice
se trovi offset:
    apri file dati
    vai a offset
    leggi record
altrimenti:
    record assente

La parte concettuale importante non e il linguaggio, ma il fatto che l'indice evita una scansione completa del file dati.

Regola pratica: un indice ha senso solo se la chiave e stabile e il formato del file dati permette di ritrovare il record in modo univoco.
