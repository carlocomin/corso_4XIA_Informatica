CSV e TSV: scrivere e leggere

Obiettivo della pagina

Vedere il flusso base del lavoro con file testuali organizzati in righe e colonne.

Idea chiave: con CSV e TSV non leggi "record gia pronti": leggi righe di testo e poi le trasformi in campi.

Che cos'e un CSV

Un file CSV contiene righe. Ogni riga rappresenta un record e ogni campo e separato da un delimitatore.

  • nel corso useremo spesso ; come separatore CSV;
  • in un TSV il separatore e il tab;
  • la prima riga puo contenere l'header con i nomi delle colonne.

Modello operativo corretto

  1. apri il file in modalita testuale;
  2. leggi una riga alla volta;
  3. spezza la riga nei campi;
  4. converti i campi nei tipi richiesti;
  5. costruisci il record logico, per esempio una struct.

Questo schema evita di confondere lettura fisica del file e interpretazione dei dati.

Esempio minimo in C++

#include <fstream>
#include <iostream>
#include <string>

int main(){
    std::ifstream in("studenti.csv");
    if(!in){
        std::cerr << "Errore apertura\n";
        return 1;
    }

    std::string riga;
    while(std::getline(in, riga)){
        std::cout << riga << "\n";
    }
}

Qui stiamo ancora leggendo solo righe. La divisione in colonne arriva dopo.

Scrittura: la logica e simmetrica

Per scrivere un CSV fai il percorso opposto:

  1. prendi il record logico;
  2. trasformi i campi in testo;
  3. li unisci con un separatore;
  4. scrivi la riga nel file.

Questa simmetria tra scrittura e lettura sara importante nella prossima slide, quando parleremo di round-trip.

Errore tipico: usare operator>> come se il CSV fosse una semplice sequenza di valori separati da spazi. Con i file a colonne conviene ragionare per righe e parsing esplicito.
