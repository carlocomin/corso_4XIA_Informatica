Read e write in C++ e C

Obiettivo della pagina

Vedere come le primitive di lettura e scrittura binaria implementano il modello visto nella slide precedente.

Idea chiave: read/write in C++ e fread/fwrite in C non "capiscono" i dati: copiano byte tra file e memoria.

Esempio base in C++

#include <fstream>
#include <cstdint>

int main(){
    std::ofstream out("dati.bin", std::ios::binary);
    if(!out) return 1;

    std::uint32_t n = 3;
    int valori[3] = {10, 20, 30};

    out.write((const char*)&n, sizeof n);
    out.write((const char*)valori, sizeof valori);
}

Qui scriviamo prima il numero di elementi, poi il blocco dati. E gia una forma minima di formato esplicito.

Esempio base in C

#include <stdio.h>
#include <stdint.h>

int main(void){
    FILE* f = fopen("dati.bin", "wb");
    if(!f) return 1;

    uint32_t n = 3;
    int valori[3] = {10, 20, 30};

    fwrite(&n, sizeof n, 1, f);
    fwrite(valori, sizeof(int), 3, f);
    fclose(f);
}

La logica e identica: apri in binario, scrivi header o conteggi, poi scrivi i dati.

Che cosa bisogna controllare

  • l'apertura del file deve riuscire;
  • la quantita di byte o elementi letti deve essere quella attesa;
  • la lettura deve seguire esattamente lo stesso ordine usato in scrittura.

Se inverti l'ordine dei campi o cambi un tipo, il file non si interpreta piu nello stesso modo.

Che cosa non fare

  • non usare queste funzioni su oggetti con puntatori interni come std::string;
  • non mescolare testo e binario nello stesso file senza un formato chiaro;
  • non dimenticare std::ios::binary o la modalita "b" quando serve.
Errore tipico: pensare che se una lettura non da errore allora il formato sia corretto. Un file puo essere letto senza errori formali ma essere interpretato in modo sbagliato.
