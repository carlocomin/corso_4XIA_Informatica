Seek e cursore nei file binari

Obiettivo della pagina

Capire perche i file binari permettono accesso casuale naturale e come si esprime questa idea in C++ e in C.

Idea chiave: ogni stream mantiene una posizione corrente. Con i seek puoi spostarti direttamente all'offset che ti serve senza rileggere tutto il file.

Il modello del cursore

Immagina il file come una sequenza di byte numerati da 0 in poi. Il cursore dice "dove sei adesso".

  • con tellg, tellp o ftell chiedi la posizione corrente;
  • con seekg, seekp o fseek ti sposti altrove;
  • questo funziona bene quando il formato usa lunghezze note o offset ben definiti.

Esempio in C++

std::ifstream in("dati.bin", std::ios::binary);
in.seekg(0, std::ios::end);
auto size = in.tellg();
in.seekg(0, std::ios::beg);

Il pattern piu semplice e: vai in fondo, misura, torna all'inizio.

Esempio in C

FILE* f = fopen("dati.bin", "rb");
fseek(f, 0L, SEEK_END);
long size = ftell(f);
fseek(f, 0L, SEEK_SET);

Ancora una volta la logica e identica: cambia solo la superficie dell'API.

Quando il seek diventa potente

  • record tutti della stessa dimensione;
  • header iniziale che contiene offset o conteggi;
  • indice separato con coppie chiave - offset.

In questi casi puoi andare quasi subito al punto giusto del file.

Regola pratica: il seek ha senso solo se il formato ti dice dove andare. Senza un contratto preciso, l'offset e solo un numero senza significato.
