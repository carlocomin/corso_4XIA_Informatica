Che cos'e un file

Obiettivo della pagina

Fissare un modello mentale corretto di file prima di parlare di codice, stream e funzioni di libreria.

Idea chiave: un file non e "memoria salvata", ma una sequenza di byte a cui diamo significato tramite regole di formato.

Tre livelli da tenere distinti

LivelloDomandaEsempio
LogicoChe cosa rappresentano questi byte?CSV, immagine PNG, archivio binario
Sistema operativoCome apro, leggo e chiudo il file?path, permessi, handle, stream
FisicoDove stanno davvero i dati?blocchi su SSD o disco

Nel corso lavoriamo quasi sempre sui primi due livelli. Il terzo ci interessa solo come idea generale.

Che cosa contiene davvero un file

Dal punto di vista del sistema operativo, un file contiene byte. Il significato di quei byte dipende dal formato scelto.

  • se il formato e testuale, i byte rappresentano caratteri;
  • se il formato e binario, i byte rappresentano campi, header, record o blocchi definiti dal programma;
  • se manca una regola chiara, il file puo esistere ma non essere interpretabile in modo affidabile.

Path, working directory e permessi

Per usare un file non basta che "esista". Serve anche trovarlo e avere il diritto di aprirlo.

  • Percorso assoluto: parte dalla radice del filesystem.
  • Percorso relativo: parte dalla cartella di lavoro del programma.
  • Permessi: un file puo essere leggibile ma non scrivibile, oppure bloccato da un errore di percorso.

Nelle esercitazioni conviene usare file nella stessa cartella del programma o in sottocartelle semplici.

Errore tipico: pensare che il file sia solo "un contenitore neutro". In realta ogni file funziona bene solo se chi scrive e chi legge condividono le stesse regole.
